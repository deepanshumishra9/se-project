Implementation Plan: Bibi Reader Integration
This plan outlines the steps to integrate the Bibi EPUB Reader into the existing project. This will allow users to read books directly in the browser while syncing their progress to the database.

User Review Required
IMPORTANT

Library Files: Bibi is a large core library that shouldn't be fully generated by an AI in a single turn. I will provide the directory structure and the custom configuration files. You will need to download the Bibi core library and place it in the /public/bibi folder.

Download Bibi from GitHub Releases.
Extract the contents into your project at public/bibi/.
WARNING

PDF Support: Bibi is natively an EPUB/HTML reader. It does not support PDF books. For PDF books, the application will fall back to the browser's native PDF viewer or a direct download.

Open Questions
Book Sourcing: Since we use the Gutenberg API, books are hosted externally. Bibi performs best with local files. Should I implement a server-side "proxy" to fetch the EPUB to the local bibi-bookshelf when a user starts reading, or should we try to load the Gutenberg URL directly (subject to CORS)?
Proposed Solution: I will implement a "Fetch & Read" logic that optionally downloads the file to the server for a smoother Bibi experience.
